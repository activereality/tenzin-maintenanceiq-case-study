# Pilot Success Criteria

Use these criteria as goals to validate during a focused Tenzin MaintenanceIQ pilot. They are not guaranteed outcomes; the pilot should confirm whether the product is useful for the selected facility, assets, documents, and workflows.

| Area | What To Validate | Example Evidence |
| --- | --- | --- |
| Retrieval accuracy | Questions return relevant source material for the selected asset or facility context. | Sample troubleshooting questions produce source-backed answers from the expected manuals or procedures. |
| Citation usefulness | Citations help users confirm the answer against the original document. | Technicians or managers can open cited manual pages and judge whether the references are useful. |
| Time saved finding documentation | The pilot reduces time spent searching PDFs, binders, shared drives, or messages. | Before/after lookup notes, timed demo tasks, or user feedback from common troubleshooting scenarios. |
| Technician usability | Technicians can ask plain-language questions and understand the response format. | Pilot users can complete common questions without heavy coaching after initial orientation. |
| Asset/document coverage | The selected assets have enough approved documentation to support meaningful retrieval. | A defined starter set of assets has manuals, procedures, drawings, or troubleshooting references uploaded and linked. |
| Feedback loop | Users can flag helpful or unhelpful answers and references. | Feedback is captured during pilot reviews and used to identify gaps in documents or answer quality. |
| Security/data comfort | Stakeholders are comfortable with the pilot data boundary and document handling approach. | Approved document scope, user access, and account/facility boundaries are reviewed before expansion. |
| Go/no-go expansion decision | The team can make a clear decision about whether to broaden scope. | Stakeholders agree to expand, revise scope, pause, or stop based on documented pilot findings. |

## Suggested Pilot Review Questions

- Which questions produced useful, source-backed answers?
- Which citations were helpful, missing, or confusing?
- Which documents or assets need cleanup before broader use?
- Did technicians trust the workflow enough to use it during real troubleshooting?
- What data, access, or review requirements must be addressed before expansion?
