# 01 — Product Overview

## Product summary

Tenzin MaintenanceIQ is a SaaS platform for industrial maintenance knowledge, troubleshooting, and machine documentation. The product is designed around a simple idea: maintenance teams should be able to ask questions about a machine, receive answers grounded in cited documentation, and preserve the results of troubleshooting work as institutional knowledge.

The initial focus is not a traditional CMMS replacement. The MVP is intentionally centered on machine knowledge, document search, AI-assisted troubleshooting, issue capture, citations, and asset context.

## Target users

Primary users include:

- Maintenance technicians
- Controls engineers
- Plant engineers
- Reliability teams
- Machine builders and integrators
- Operations teams supporting complex equipment

## Core user problems

Industrial teams often struggle with fragmented knowledge:

- Manuals are stored in different folders, binders, drives, or inboxes.
- PLC/HMI/VFD program backups are hard to find during downtime.
- Experienced operators and technicians carry undocumented tribal knowledge.
- Troubleshooting notes are not consistently connected to the machine that produced the issue.
- Search results are often generic and not grounded in the specific machine documentation.

## MVP product focus

The MVP is built around these foundational concepts:

| Area | Purpose |
| --- | --- |
| Assets / Machines | Central context for documentation, components, issues, and AI assistance |
| Machine Components | PLCs, HMIs, VFDs, motors, sensors, safety systems, networks, and other machine parts |
| Documents | Manuals, drawings, procedures, diagrams, and uploaded reference material |
| Embeddings/Search | Chunked machine-specific knowledge retrieval |
| Chat | AI-assisted troubleshooting and knowledge access |
| Citations | Cited references so answers remain auditable |
| Feedback | User feedback on answer quality and citation usefulness |
| Issues | MaintenanceIQ issue capture and timeline history |
| Public Identifier / QR | Fast machine lookup from the plant floor |
| Program Loads | Future disaster recovery repository for PLC/HMI/VFD code backups |

## Product principle

Tenzin should make machine knowledge easier to find, trust, and reuse. AI is useful only when it is grounded in the plant's actual documentation and workflow context.
